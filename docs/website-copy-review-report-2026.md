全体として、前回のレビュー（`top-page-copy-review.md`）で「実装優先度：すぐ直したいもの」に挙げられていた、Program内の「（仮）」表記の削除や、Footerの「Sample sound map」の削除などは適切に反映されており、公開サイトとしての完成度が向上しています。

## 3. `index.html` の新規不整合・改善推奨項目

### ① ページ構造とアンカーリンクの不整合
*   **問題点**:
    *   Headerナビゲーションには「About」「Sound Map」「Program」「Join」がありますが、メインイベントの紹介セクションである「**Event**（#event）」へのリンクがありません。
    *   Heroエリアの「詳細を見る」ボタン（141行目）の遷移先が `#program`（今年度の予定）になっています。しかし、イベントの詳細が最も詳しく書かれているのは `#event`（177行目～）であるため、クリックしたユーザーが詳細情報をスキップして下方に飛んでしまいます。
*   **推奨案**:
    *   Heroの「詳細を見る」の遷移先を `#event` に変更する。
    *   Headerの「Program」リンクの遷移先を `#program` のままにするか、ナビゲーション構成を見直す。

### ② 集合場所の「徒歩分数」の不一致
*   **問題点**:
    *   199行目（Event facts）: `まちラボおおみや（JR大宮駅東口より徒歩1分）`
    *   仕様書（見積書）: `まちラボおおみや（JR大宮駅徒歩3分）`
    *   大宮ラクーン（まちラボおおみやの入るビル）は東口から徒歩1～2分程度ですが、表記に2分のズレがあります。
*   **推奨案**:
    *   ユーザーに最も近い案内として `徒歩1分` または `徒歩2分` に統一する。

### ③ 構造化データ（JSON-LD）内の Performer 表記
*   **問題点**:
    *   84～86行目の JSON-LD 内で、Performer が次のように記述されています。
        ```json
        {
          "@type": "Person",
          "name": "Minuma / Nukumori Eugene"
        }
        ```
        スラッシュ区切りで書かれているため、検索エンジン等のクローラーが「二人の人物」なのか「別名を持つ一人の人物」なのか判断しづらく、構造化データとして不自然です。
*   **推奨案**:
    *   もし同一人物の別名であれば、芸名または本名に統一するか、以下のように `alternateName` を使用して整理します。
        ```json
        {
          "@type": "Person",
          "name": "Nukumori Eugene",
          "alternateName": "Minuma"
        }
        ```

## 4. `sounds.geojson` の文言チェック＆改善推奨項目

音地図のデータである `sounds.geojson` の `properties` 内に、以下の表記揺れや改善点が確認されました。

### ① スポットタイトル（`title`）に開発用データが露出している
*   **問題点**:
    七里や岩槻のスポットにおいて、タイトルに日付と記録者イニシャル（例: `20260405_NK`）がそのまま入っており、地図のピンをクリックした際に画面上でそのまま表示されてしまいます。
    *   `大圓寺（七里）20260405_NK` (216行目)
    *   `七里駅の三本櫻（七里）20260405_NK` (242行目)
    *   `小御嶽神社（岩槻）20260426_FKD` (268行目)
*   **推奨案**:
    *   タイトルはシンプルに `大圓寺（七里）`、`七里駅の三本櫻`、`小御嶽神社（岩槻）` とし、日付や記録者情報は `note`（録音メモ）の方に移動します。

### ② 録音日付・時間の表記形式の揺れ
*   **問題点**:
    録音日時の記述方法がスポットによってバラバラです。
    *   ルミネ周辺: `録音日時: 2024.10.06 19:30` (ドット区切り、時間あり)
    *   連絡通路: `録音日: 2024.10.05頃。` (「頃」付き、時間なし)
    *   七里・岩槻周辺: `録音日: 2026.04.05` (ドット区切り、時間なし)
    *   曳舟・千住周辺: `録音日時: 2024.06.09 14:51頃` (ドット区切り、時間あり、「頃」付き)
    *   進修館周辺: `録音日時: 2024.11.10 16:06頃。` (末尾に句点あり)
    *   大宮公園動物園・まめの木・鴻沼・ほこすぎ橋: 日付情報の記載なし
*   **推奨案**:
    *   書式を `録音日時: YYYY.MM.DD HH:MM頃` または `録音日: YYYY.MM.DD` に統一します。
    *   末尾の句点（「。」）の有無も統一します。

### ③ 記録者およびレコーダー（機材）情報の表記揺れと欠落
*   **問題点**:
    記録者名や使用機材の英語・日本語が混在しており、また情報が抜けているスポットもあります。
    *   ルミネ周辺: `Recorder: Zoom H1 essential (eugen.n)`
    *   七里・岩槻周辺: `記録者表記: NK` (日本語表現、機材なし)
    *   曳舟・千住周辺: `Recorder: H1 essential + Clippy EM272`
    *   進修館周辺: 機材・記録者情報ともに記述なし
*   **推奨案**:
    *   機材紹介を `録音機材: Zoom H1 essential` または `Recorder: Zoom H1 essential` で統一します。
    *   記録者は `録音: eugen.n` や `Recorded by NK` など、表記ルールを定めて統一します。

### ④ 漢字の表記揺れ（櫻 vs 桜）
*   **問題点**:
    *   スポットタイトルおよび説明文に `三本櫻` (旧字) と `桜` (新字) が混在しています。
    *   242行目: `"title": "七里駅の三本櫻"`
    *   246行目: `駅前の桜`
*   **推奨案**:
    *   SoundCloudの曲名が「三本櫻」である場合はタイトルにそれを残しつつも、説明文内では `三本櫻` か `三本桜` のどちらかに表記を寄せた方が自然です。

### ⑤ さいたま市外のスポットに関する説明の補足
*   **問題点**:
    *   「さいたまの音地図」というセクションでありながら、東京都墨田区（曳舟）や足立区（千住）、埼玉県宮代町（進修館）の音源が含まれています。これらについて、なぜさいたま以外の場所の音があるのかという文脈が初見のユーザーには伝わりにくい状態です。
*   **推奨案**:
    *   「さいたま周辺やメンバーの活動地域での記録」であることが伝わるよう、セクション導入文やスポット説明文に一言補足を加える（例: `tags` に `sound-journey` や `遠征録音` などのタグを付与する、あるいは説明文に「メンバーの活動エリアでの記録」と書き添える）。
